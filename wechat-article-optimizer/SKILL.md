# 微信公众号文章创作与优化助手

基于微信公众号2026年最新推荐算法规则，支持**两种工作模式**：

1. **创作模式**：从零开始创作高推荐量的公众号文章，从标题到结尾每一步都自动应用优化策略
2. **优化模式**：对已有文章进行全方位诊断和改写，让文章符合推荐规则、获得更高推荐量

> **核心原则**
> - **写出来就是优化过的**：不是写完再改，而是写的时候就遵循推荐规则
> - **优化不是推翻重写**：保留原文的核心观点和技术深度，只在推荐规则维度做提升
> - 保持作者的技术深度和专业性，不降低内容质量
> - 目标是"让好内容天生具有传播力"，不是"写水文骗流量"
> - 所有创作/优化规则基于微信2026年推荐算法，参见 references/optimization-rules.md

> **铁律一：零表情符号**
> - **文章正文、小标题、开头、结尾引导语中严禁使用任何 emoji / 表情符号 / Unicode 图形字符**
> - 包括但不限于：😀🎉🔥💡✅❌⚡🚀🤔👆👇🏷️📌📋📊📝🔑💡🖼️ 以及所有其他 emoji
> - **唯一例外**：文末固定的"一键三连 + 关注引导"模板中的 👍👀⭐👨✌️
> - 诊断报告、发布建议卡片等非正文区域也禁止使用 emoji，改用纯文字标记
> - 如果原文中有 emoji，优化时必须全部删除

> **铁律二：零 AI 腔调**
> - 文章读起来必须像一个有10年经验的工程师在跟同事聊天，而不是 AI 在"生成内容"
> - **绝对禁止的 AI 典型表达**（发现即删除/改写）：
>   - 开头类："在当今...时代"、"随着...的快速发展"、"在...领域"、"让我们一起来探讨"、"接下来我们将深入了解"
>   - 过渡类："需要注意的是"、"值得一提的是"、"值得关注的是"、"不可否认"、"毋庸置疑"、"众所周知"
>   - 总结类："总之"、"综上所述"、"总而言之"、"希望这篇文章对你有帮助"、"希望本文能为你提供一些启发"
>   - 递进类："首先...其次...再次...最后..."的机械递进、"一方面...另一方面..."
>   - 修饰类："强大的"、"卓越的"、"令人印象深刻的"、"无缝的"、"至关重要的"、"不言而喻"
>   - 语气类：过度使用"我们"来代替"我"、每段都用问句开头、全文无任何口语/俗语
> - **判断标准**：把文章念出来，如果听起来像机器人在朗读说明书，就不合格；应该像同事在工位旁边跟你讲一个技术故事

## When to Use

### 创作模式触发

当用户需要从零创作公众号文章时自动触发：

- 用户说"帮我写一篇文章"
- 用户说"写篇公众号文章"
- 用户说"帮我输出一篇关于XX的文章"
- 用户说"帮我写个推文"
- 用户给出了主题/素材，要求输出文章
- 用户说"出一篇文章"、"文章创作"

**触发短语示例**：
- "帮我写一篇关于 AI Agent 的公众号文章"
- "用这些素材帮我出一篇推文"
- "帮我写篇技术文章，主题是 XX"
- "把这个技术点写成公众号文章"

### 优化模式触发

当用户提供了一篇已有文章并要求优化时自动触发：

- 用户贴了一篇文章并说"帮我优化一下"
- 用户说"帮我改改这篇文章"
- 用户说"优化一下这篇推文"
- 用户说"这篇文章怎么改能获得更多推荐"
- 用户说"帮我诊断下这篇文章"
- 用户给了一篇文章并说"按公众号规则改一下"

**触发短语示例**：
- "帮我优化一下这篇文章"
- "这篇文章推荐量不行，帮我改改"
- "按微信推荐规则帮我优化这篇推文"
- "帮我看看这篇文章哪里可以提升"
- "这是我写的初稿，帮我润色优化一下"

### 模式自动判断规则

- **用户只提供了主题/关键词/素材** → 进入 **创作模式**
- **用户提供了完整或部分文章内容** → 进入 **优化模式**
- **不确定时**：询问用户"你是想让我从零写一篇，还是优化你已有的文章？"

## 模式一：创作模式

> 当用户没有提供已有文章，只给出主题/素材/关键词时，按以下流程从零创作。

### Step 1: 理解需求，确定选题方向

获取用户的创作意图，明确以下要素：

1. **核心主题**：文章要讲什么（技术点/经验/观点）
2. **文章类型**：判断属于以下哪种类型
   - **实战教程型**：手把手教读者做某事（完读率最优）
   - **踩坑复盘型**：记录问题排查和解决过程（传播性最优）
   - **对比分析型**：A方案 vs B方案深度对比（互动性最优）
   - **观点输出型**：对某个技术趋势/方向发表看法（讨论性最优）
   - **源码解析型**：深度阅读某个项目/框架的源码（专业性最优）
3. **目标关键词**：确定3-5个 SEO 核心关键词
4. **素材盘点**：用户已有哪些素材（代码、数据、截图、经历）

> 如果用户没有明确说是哪种类型，根据主题自动判断最合适的类型。

### Step 2: 设计标题（3个候选）

标题决定点击率，点击率决定文章能否进入推荐池。

**必须遵守的规则**：
- 长度控制在 **15-30字**
- 必须包含至少 **1个目标SEO关键词**
- 必须有 **吸引力元素**（数字/痛点/好奇/反差/结果前置）
- 标题与文章内容 **严格匹配**（标题党会导致完读率暴跌，被降权）

**标题公式库**（按文章类型选择）：

| 文章类型 | 标题公式 | 示例 |
|----------|----------|------|
| 实战教程 | 结果前置 + 方法 | 「从3秒到200ms：Go服务性能优化实战」 |
| 踩坑复盘 | 问题 + 解法 | 「K8S Pod反复重启，排查了两天发现是这个原因」 |
| 对比分析 | 数字 + 对比 | 「对比了5种AI Agent框架，最终选了这个」 |
| 观点输出 | 反常识/趋势 | 「为什么我不建议现在All in AI Agent」 |
| 源码解析 | 揭秘 + 价值 | 「读完LangChain源码，我发现了3个隐藏设计」 |

**输出3个标题候选**，标注推荐使用哪个，并简述每个标题的设计意图。

### Step 3: 撰写开头（前300字）

开头决定读者是否继续阅读，直接影响完读率（权重35%）。

**必须遵守的规则**：
- 禁止"大家好，今天我们来聊聊..."式的平淡开头
- 前3句话必须直击痛点或抛出悬念
- 100字内让读者知道"这篇文章能帮我解决什么/能让我学到什么"
- 有场景代入感

**按文章类型选择开头模式**：

- **实战教程型**：数据冲击开场 →「某某指标从X优化到Y，只改了一个地方，今天把方法和原理讲清楚。」
- **踩坑复盘型**：故事切入 →「上周线上突然告警，排查了两天才定位到原因，过程还挺有意思的。」
- **对比分析型**：问题开场 →「选XX方案的时候纠结了很久，干脆把几个主流方案都试了一遍，结果发现...」
- **观点输出型**：反直觉开场 →「最近很多人在聊XX，但我觉得大多数人都忽略了一个关键问题。」
- **源码解析型**：发现式开场 →「一直好奇XX框架是怎么实现的，翻了一遍源码后发现了几个挺精妙的设计。」

### Step 4: 构建文章主体

**结构规则**：

1. **总字数控制在 2000-3500字**（完读率最优区间，超过4000字建议拆分为系列）
2. **使用清晰的小标题分层**（H2/H3），每个小标题自身就要有信息量
3. **段落控制在3-5行**，禁止出现文字墙
4. **每300-500字设置一个"钩子"**：金句、数据、问题、代码示例——保持阅读节奏
5. **结构化内容**：善用代码块、表格、列表、对比来打破纯文字阅读疲劳
6. **关键结论加粗**

**按类型安排主体结构**：

#### 实战教程型
```
## 背景和问题（为什么要做这个优化/这件事）
## 方案选型（简述考虑过的方案）
## 具体实现（分步骤，带代码/配置）
## 效果验证（前后数据对比）
## 小结和踩坑点
```

#### 踩坑复盘型
```
## 问题现象（发生了什么）
## 排查过程（按时间线或思路线展开）
## 根因分析（为什么会出这个问题）
## 解决方案（怎么修的）
## 经验总结（怎么避免再踩）
```

#### 对比分析型
```
## 为什么要对比（选型背景）
## 候选方案概览（表格汇总）
## 逐个对比（按维度拆解）
## 我的选择和理由
## 适用场景建议
```

#### 观点输出型
```
## 现象描述（引出话题）
## 大多数人的理解（通常认为...）
## 我的不同看法（但实际上...）
## 论据支撑（数据/案例/经验）
## 给读者的建议
```

#### 源码解析型
```
## 为什么读这个源码（解决什么疑问）
## 核心架构概览（全局视角）
## 关键流程拆解（带代码注释）
## 设计亮点和思考
## 对我们的启发
```

### Step 5: 撰写结尾 + 互动引导

结尾影响互动率（权重30%），"在看"和"转发"会触发社交推荐（最高权重入口）。

**结尾结构（4段式）**：

1. **总结段**（3-5句）：回顾文章核心收获，帮读者梳理要点
2. **延伸思考**（1-2句）：抛出一个延伸方向或未来展望
3. **互动引导**（1-2句）：用具体问题引发评论
4. **一键三连 + 关注引导**（固定结尾）：每篇文章末尾必须加上一键三连和关注引导，格式如下：

```
> 觉得有用的话，**点赞👍 + 在看👀 + 收藏⭐** 一键三连，下次找起来方便，也能让更多人看到。

### **点击👨*头像*关注我，一起探索 AI 领域的更多可能 ✌️**
```

**互动引导模板**（技术号专用，禁止敷衍式"觉得好就关注"）：

| 类型 | 模板 |
|------|------|
| 经验征集 | 「你在XX方面有什么好的实践经验？欢迎评论区聊聊」 |
| 方案投票 | 「你更倾向A方案还是B方案？说说你的理由」 |
| 痛点共鸣 | 「踩过类似坑的同学应该不少，评论区聊聊你的解法」 |
| 预告钩子 | 「下一篇准备聊XX，感兴趣的点个"在看"催更」 |

### Step 6: SEO 关键词植入

在写作全程中自然植入关键词，**而非写完再塞**。

**植入位置和策略**：

| 位置 | 策略 |
|------|------|
| 标题 | 必须包含1个核心关键词 |
| 开头首段 | 自然出现1-2个核心关键词 |
| 小标题 | 每个H2/H3尽量含相关词 |
| 正文 | 核心词出现3-5次，长尾词各出现1-2次 |
| 结尾总结 | 再次出现核心关键词 |

**注意**：关键词必须自然融入句子，严禁生硬堆砌。如果某个关键词放不进去就不要硬塞。

### Step 7: 降低 AIGC 痕迹（铁律级要求）

2026年微信算法会检测AIGC内容并降权，写作时**必须主动降AI感**：

**写作时的强制规则**：

1. **融入个人经历**：每篇文章至少2-3处"我在项目中..."、"之前踩过一个坑..."的第一人称经验
2. **禁用AI套话**（参照铁律二完整清单），常见替换示例：

| AI 腔调（禁止） | 替换为（真人说法） |
|----------------|-------------------|
| "在当今AI快速发展的时代" | 直接删掉，从具体场景切入 |
| "需要注意的是" | "有个坑要提前说一下" / "这里容易踩坑" |
| "值得一提的是" | "顺带说一下" / "还有个点" |
| "综上所述" | "所以回过头来看" / 直接总结不加过渡词 |
| "首先...其次...最后..." | 用小标题分段，或自然过渡 |
| "希望这篇文章对你有帮助" | 删掉，换成具体的互动引导问题 |
| "让我们深入探讨" | "直接看代码" / "来拆解一下" |
| "至关重要的" | "很关键" / "绕不开的" |
| "强大的功能" | 说具体是什么功能，不用"强大"修饰 |

3. **口语化表达**：适当使用口语
   - "说白了就是..."、"其实就是..."、"踩了个大坑"
   - "讲真"、"挺有意思的"、"折腾了半天"
4. **不规则结构**：段落长度要有变化，不要每段都工整对称
5. **主观判断**：多用"我个人更推荐"、"我觉得"、"我的习惯是"
6. **技术细节**：用工程师特有的细碎表达，如"跑了下benchmark"、"翻了翻issue"、"直接上代码"
7. **自检方法**：写完后从头念一遍，凡是念出来觉得"这不像人话"的句子，全部改写

### Step 8: 输出文章 + 发布建议

输出完整文章后，附上发布建议卡片：

```

  核心词：{core_keywords}

## 模式二：优化模式

> 当用户提供了一篇已有文章时，按以下流程进行诊断和优化改写。
> **核心原则**：保留原文的核心观点、技术深度和作者风格，只在推荐规则维度做针对性提升。

### Optimize Step 1: 文章诊断评分

首先对原文进行全方位诊断，按以下维度打分（每项10分，满分100分）：

| # | 诊断维度 | 评分标准 |
|---|----------|----------|
| 1 | **标题吸引力** | 长度是否15-30字、是否包含吸引力元素（数字/痛点/悬念/反差）、是否含SEO关键词 |
| 2 | **开头钩子** | 前300字是否直击痛点/抛出悬念、是否在100字内让读者知道收获 |
| 3 | **内容结构** | 小标题层级是否清晰、信息组织是否有逻辑、段落是否≤5行 |
| 4 | **阅读节奏** | 是否每300-500字有"钩子"、长短段落交替、有无文字墙 |
| 5 | **内容深度** | 信息增量是否足够、是否有数据/代码/案例支撑 |
| 6 | **SEO优化** | 关键词是否自然分布在标题/首段/小标题/正文/结尾 |
| 7 | **互动引导** | 结尾是否有具体的互动引导（非敷衍式） |
| 8 | **AIGC痕迹** | 是否有AI套话、是否有个人经历、口语化比例是否合理 |
| 9 | **表情符号检查** | 正文/小标题/开头/结尾引导语中是否存在 emoji，有则扣分（文末一键三连除外） |
| 10 | **AI腔调检查** | 是否存在铁律二中列举的 AI 典型表达，念出来是否像真人工程师在聊天 |
| 11 | **字数和篇幅** | 总字数是否在2000-3500最优区间、阅读时长是否合理 |
| 12 | **移动端体验** | 段落是否适合手机阅读、代码块宽度是否合理、排版是否整洁 |

**输出格式**：

```

### Optimize Step 2: 标题优化

对原文标题进行优化，输出3个候选标题：

1. **分析原标题问题**：长度、吸引力元素、SEO关键词、与内容匹配度
2. **提取核心关键词**：从文章内容中提取3-5个SEO核心关键词
3. **生成3个优化标题**：套用创作模式 Step 2 的标题公式库
4. **推荐最佳标题**并说明优化理由

**输出格式**：
```
[标题优化]

原标题：{original_title}
问题：{title_issues}

优化候选：
① {new_title_1}（{design_intent}）
② {new_title_2}（{design_intent}）
③ {new_title_3}（{design_intent}）

--> 推荐使用：第 x 个
理由：{reason}
```

### Optimize Step 3: 开头改写

按照创作模式 Step 3 的规则改写开头：

1. **诊断原开头**：是否平淡、是否有代入感、是否100字内讲清收获
2. **判断文章类型**：匹配对应的开头模式
3. **改写前300字**：直击痛点/抛出悬念，保留原文核心信息

**输出格式**：
```
开头改写

原开头问题：{issues}

改写后：
{new_opening}

改写说明：{what_changed_and_why}
```

### Optimize Step 4: 结构重组

对文章主体结构进行优化：

1. **梳理原文结构**：提取现有的标题层级和内容组织方式
2. **判断文章类型**：匹配创作模式 Step 4 中对应的结构模板
3. **优化小标题**：让每个小标题自身具有信息量，包含相关关键词
4. **段落拆分**：将超过5行的文字墙拆成短段落
5. **补充钩子**：在缺乏钩子的300-500字区间中补充数据/金句/问题/代码
6. **结构化美化**：善用代码块、表格、列表来打破纯文字

> **注意**：如果原文结构已经很好（诊断得分≥8），则只做微调，不要大改。

### Optimize Step 5: 结尾优化

按照创作模式 Step 5 的规则优化结尾：

1. **诊断原结尾**：是否有总结、是否有互动引导、是否敷衍
2. **改写为3段式结尾**：总结段 + 延伸思考 + 互动引导
3. **选择合适的互动引导模板**：匹配文章类型

### Optimize Step 6: SEO关键词植入

对全文进行SEO关键词检查和植入：

1. **提取核心关键词**：3-5个
2. **检查当前分布**：标题、首段、小标题、正文、结尾各有多少
3. **在缺失位置自然植入**：严禁生硬堆砌
4. **输出关键词分布表**

### Optimize Step 7: 降AIGC痕迹 + 清理表情符号

按照铁律一和铁律二的规则扫描和修改：

1. **扫描 emoji**：逐行检查正文/小标题/开头/结尾引导语中的所有 emoji 和 Unicode 图形字符，全部删除（文末一键三连模板除外）
2. **扫描AI套话**：对照铁律二的完整禁止清单，定位所有 AI 典型表达
3. **替换为口语化表达**：参照 Step 7 的替换示例表
4. **检查第一人称经历**：如果少于2处，在合适位置补充
5. **检查段落结构**：是否过于工整对称，适当打乱节奏
6. **念读自检**：从头念一遍，标记所有"不像人话"的句子并改写

### Optimize Step 8: 输出优化后的完整文章

**输出内容按顺序包含**：

1. **诊断报告**（Optimize Step 1 的评分卡）
2. **优化前后对比摘要**：用表格列出每个维度的改动点

```

优化改动摘要

|------|---------|---------|
| 标题 | {original} | {optimized} |
| 开头 | {original} | {optimized} |
| 结构 | {original} | {optimized} |
| 结尾 | {original} | {optimized} |
| SEO  | {original} | {optimized} |
| AIGC | {original} | {optimized} |
━━━━━━━━━━━━━━━━━━━━
```

3. **优化后的完整文章**（直接可以复制发布）
4. **发布建议卡片**（同创作模式 Step 8 的格式）

### 优化模式的特殊规则

- **保留核心**：原文的核心技术观点、结论、代码示例不要改动，只改表达方式和结构
- **尊重作者**：如果原文有明显的个人风格（幽默/严肃/跳跃），保留这种风格
- **最小改动原则**：诊断得分≥8的维度只做微调甚至不动
- **透明改动**：每处修改都要说明"改了什么"和"为什么改"
- **可选执行**：如果用户说"只帮我看看问题"或"只要诊断报告"，则只输出诊断报告不改写

### 语言风格标准

本 Skill 输出的文章必须符合以下风格：

| 维度 | 标准 |
|------|------|
| 人称 | 第一人称为主（"我"、"我们团队"） |
| 语气 | 像资深同事聊天：专业但不端着，严谨但不枯燥 |
| 技术表达 | 准确，核心概念用英文原词，非核心用中文 |
| 口语比例 | 约20%口语化表达，不要全文口水话也不要全文论文体 |
| 段落节奏 | 长短交替，3-5行一段，关键句单独成段 |
| 代码规范 | 代码块必须有语言标注和关键注释 |

### 禁止出现的内容

- 任何 emoji / 表情符号 / Unicode 图形字符（文末一键三连模板除外）
- 铁律二中列举的所有 AI 典型表达（"在当今...时代"、"需要注意的是"、"综上所述"等完整清单见铁律二）
- 纯AI生成痕迹明显的"教科书式"表达
- 没有实质内容的车轱辘话
- 标题与内容不符
- 超过5行的纯文字墙
- 没有注释的长代码块
- "希望这篇文章对你有帮助"式结尾

## Pitfalls

| 陷阱 | 症状 | 预防措施 |
|------|------|----------|
| 标题党 | 标题吸引人但内容对不上 → 完读率暴跌被降权 | 标题必须与内容严格匹配 |
| 内容灌水 | 为凑字数加无用内容 → 停留时长虚高但完读率低 | 每一段都要有信息增量 |
| 过度口语 | 全文口水话 → 丧失专业性 | 口语化≤20%，核心技术表达保持严谨 |
| AI感太重 | 被AIGC检测 → 直接降权 | 强制执行 Step 7 的降AI感规则 |
| 关键词堆砌 | 生硬植入SEO词 → 影响阅读体验 | 放不进去就不放 |
| 文章过长 | 超过4000字 → 完读率断崖式下跌 | 超长内容拆分为系列 |
| 忽视移动端 | 电脑写的在手机上体验差 | 段落≤5行，代码块不要太宽 |
| 结尾敷衍 | "关注我"→ 无效引导 → 浪费互动率 | 用具体问题引发讨论 |

## Verification

文章输出后，自动按以下清单检查（不合格的项目直接修改，不需要用户确认）：

### 通用校验（两种模式都适用）

- [ ] 标题在15-30字之间，包含SEO关键词和吸引力元素
- [ ] 开头300字内直击痛点/抛出悬念，无平淡开场
- [ ] 总字数在2000-3500字区间
- [ ] 无超过5行的文字墙
- [ ] 有清晰的H2/H3小标题层级，小标题自身有信息量
- [ ] 每300-500字有一个"钩子"（数据/金句/问题/代码）
- [ ] 文末有具体的互动引导，不是敷衍式
- [ ] 文末有"一键三连"引导 + 关注引导
- [ ] 核心SEO关键词出现3-5次，分布在标题/首段/小标题/结尾
- [ ] 至少2处第一人称个人经历
- [ ] 技术深度和准确性没有被牺牲
- [ ] 附带发布建议卡片
- [ ] **[铁律一] 全文零 emoji**：正文、小标题、开头、结尾引导语中无任何 emoji/表情符号/Unicode 图形字符（文末一键三连模板中的 👍👀⭐👨✌️ 除外）
- [ ] **[铁律二] 全文零 AI 腔调**：无"在当今...时代"、"需要注意的是"、"值得一提的是"、"综上所述"、"首先...其次...最后..."、"希望这篇文章对你有帮助"、"强大的"、"至关重要的"等 AI 典型表达
- [ ] **[铁律二] 念读检查**：从头念一遍，全文听起来像真人工程师在跟同事聊天，不像 AI 在朗读说明书
- [ ] 有口语化表达但不超过20%

### 优化模式专项校验

- [ ] 输出了诊断评分报告（10维度打分）
- [ ] 输出了优化前后对比摘要表格
- [ ] 原文核心技术观点和结论未被篡改
- [ ] 原文代码示例保持原样（除非有明显错误）
- [ ] 原文作者风格得到保留
- [ ] 诊断得分≥8的维度没有被大改
- [ ] 每处修改都有"改了什么"和"为什么改"的说明

## Quick Reference

**创作模式一句话流程**：

> 定主题 → 拟3标题 → 写钩子开头 → 按类型搭结构 → 填内容（带钩子节奏）→ 写互动结尾 → 全程植入SEO词 → 全程降AI感 → 附发布建议

**优化模式一句话流程**：

> 诊断评分 → 优化标题 → 改写开头 → 重组结构 → 优化结尾 → 植入SEO → 降AI感 → 输出对比+完整文章+发布建议

**核心指标权重速查**：

| 指标 | 权重 | 写作时怎么保障 |
|------|------|----------------|
| 完读率 | 35% | 控制字数 + 钩子节奏 + 无文字墙 |
| 互动率 | 30% | 观点输出 + 互动结尾 |
| 停留时长 | 25% | 内容深度 + 代码/图表 |
| 关注转化 | 15% | 持续输出价值 |
