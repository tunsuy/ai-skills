# Creator 质量检查清单

> 本文件由 Meta Creator 生成，用于确保生成的 Creator 质量达标。
> 在 Creator 交付前，逐项核对。

---

## 1. 结构完整性

### 1.1 Frontmatter
- [ ] `name` 字段存在且格式为 `skill-creator-{domain}`
- [ ] `description` 字段存在且包含触发场景
- [ ] `tools` 字段存在且至少包含 `[write, bash]`
- [ ] `generated_by` 字段存在且包含版本号

### 1.2 Phase 结构
- [ ] Phase 1（信息收集）存在且有明确轮数
- [ ] Phase 2（生成与预览）存在且有完整的生成逻辑
- [ ] Phase 3（交付）存在且有标准流程
- [ ] 各 Phase 之间有清晰的转换条件

---

## 2. 信息收集质量

### 2.1 对话设计
- [ ] 每一轮有明确的主动询问问题
- [ ] 问题自然、不像填表
- [ ] 有被动捕获逻辑（不主动问但能识别）
- [ ] 有条件追问规则（信息不足时的补充策略）

### 2.2 变量设计
- [ ] 核心变量（必须收集）不超过 6 个
- [ ] 可选变量有"有值注入 / 无值清除"逻辑
- [ ] 最后一轮有结构化汇总展示
- [ ] 包含模式推荐（轻量/完整）

---

## 3. 生成逻辑质量

### 3.1 Skill 骨架
- [ ] 启动协议完整（文件读取 + 状态恢复 + 进度声明）
- [ ] 每个阶段有：目标、动作列表、产出物、门控条件
- [ ] 流程类型正确（linear / linear_with_gate / directed_with_backtrack / parallel_convergent）
- [ ] 回退规则（如适用）定义清晰
- [ ] 并行规则（如适用）定义清晰

### 3.2 Memory 系统
- [ ] plan.md 存在且包含进度追踪
- [ ] decisions.md 存在且为追加式写入
- [ ] 领域特定记忆文件存在
- [ ] 读取优先级策略完整
  - [ ] 轻量模式：全量加载
  - [ ] 完整模式：分层优先级

### 3.3 双模式
- [ ] 轻量模式有明确定义
- [ ] 完整模式有明确定义
- [ ] 自适应规则存在（轻量加深 / 完整精简）
- [ ] 模式选择有推荐依据

### 3.4 安全规则
- [ ] 无删除文件的指令
- [ ] 无未替换的 `{{...}}` 占位符
- [ ] 文件路径均为相对路径
- [ ] 写入使用文件工具而非 Shell heredoc

---

## 4. References 文件

- [ ] 至少 2 个配套参考文件
- [ ] 检查清单覆盖所有阶段
- [ ] 复盘/总结指南结构清晰
- [ ] 领域特定模板（如有）内容实用

---

## 5. 用户体验

- [ ] 信息收集轮数 ≤ 4
- [ ] 每轮对话时间 ≤ 2 分钟
- [ ] 预览展示清晰可读
- [ ] 交付后有使用引导
- [ ] 错误处理友好（如路径冲突、文件已存在）

---

## 6. 可扩展性

- [ ] 预留自定义验收方式接口
- [ ] 结项时生成 handoff.md（跨 Creator 桥接）
- [ ] 支持多语言（至少中英文）
